Implement the Iron Crest Rolling Reserve system based on the 
official Chargebacks and Rolling Reserve Policy document.

The rolling reserve is a financial hold on rep commissions that 
protects Iron Crest against chargebacks, cancellations, and 
equipment losses. It applies to REP, LEAD, and MANAGER roles.

KEY LEGAL DISTINCTIONS FROM THE POLICY DOCUMENT:
- The rolling reserve is NOT wages — it is a conditional holdback
- Iron Crest retains final authority over reserve administration
- Reserve is held through full carrier maturity periods even after
  termination — no early release
- Governing law: State of New Jersey

  export function getMaturityDays(
    providerName: string,
    chargebackType: 'VOLUNTARY_CANCELLATION' | 'NON_PAY_DISCONNECT'
  ): number {
    const provider = providerName.toLowerCase();
    
    if (provider.includes('optimum') || provider.includes('altice')) {
      return chargebackType === 'VOLUNTARY_CANCELLATION' 
        ? 120 : 180;
    }
    if (provider.includes('astound')) {
      return 120; // both types are 120 for Astound
    }
    // Default fallback for unknown carriers
    return 180;
  }

  export function calculateMaturityDate(
    dateSold: string,
    providerName: string,
    chargebackType: 'VOLUNTARY_CANCELLATION' | 'NON_PAY_DISCONNECT'
      = 'VOLUNTARY_CANCELLATION'
  ): Date {
    const days = getMaturityDays(providerName, chargebackType);
    const saleDate = new Date(dateSold);
    saleDate.setDate(saleDate.getDate() + days);
    return saleDate;
  }

  export function isOrderMature(
    dateSold: string,
    providerName: string
  ): boolean {
    // An order is mature when ALL possible maturity periods 
    // have expired. Use the longest period for the carrier.
    const longestMaturity = calculateMaturityDate(
      dateSold, 
      providerName, 
      'NON_PAY_DISCONNECT'
    );
    return new Date() > longestMaturity;
  }

FUNCTION 2: calculateWithholding

  export function calculateWithholding(
    netCommissionCents: number,
    reserve: RollingReserve
  ): number
  
  Logic:
    if (reserve.status === 'AT_CAP') return 0;
    if (reserve.status === 'HELD') return 0;
    if (reserve.status === 'RELEASED') return 0;
    
    const withholdingPercent = reserve.withholdingPercent / 100;
    const rawWithholding = Math.floor(
      netCommissionCents * withholdingPercent
    );
    
    // Only withhold enough to reach the cap
    const remainingToCapCents = 
      reserve.capCents - reserve.currentBalanceCents;
    
    return Math.min(rawWithholding, remainingToCapCents);

FUNCTION 3: applyWithholding

  export async function applyWithholding(
    userId: string,
    salesOrderId: string,
    payStatementId: string,
    netCommissionCents: number,
    txDb?: TxDb
  ): Promise<{
    withheldCents: number,
    newBalance: number,
    newStatus: string,
    capReached: boolean,
  }>
  
  Called during pay stub generation for each order.
  
  Steps:
  1. Load or create reserve for user
  2. If reserve is AT_CAP, HELD, or RELEASED: return 0
  3. Calculate withholding amount
  4. Create reserve_transaction record:
       type: 'WITHHOLDING'
       amountCents: withheldCents
       isCredit: true
       balanceAfterCents: newBalance
       salesOrderId
       payStatementId
       description: `15% withholding from order 
                     ${invoiceNumber} commission 
                     of $${(netCommissionCents/100).toFixed(2)}`
  5. Update rolling_reserves:
       currentBalanceCents += withheldCents
       totalWithheldCents += withheldCents
       status = newBalance >= capCents ? 'AT_CAP' : 'ACTIVE'
  6. Update salesOrders:
       reserveWithheldCents = withheldCents
       reserveWithholdingApplied = true
  7. If cap just reached: create reserve_transaction 
     with type 'CAP_REACHED' and description:
     "Rolling reserve cap of $2,500.00 reached. 
      Withholding paused."
  8. Log to audit_logs:
     action: 'RESERVE_WITHHOLDING_APPLIED'
  9. Return result

FUNCTION 4: applyChargeback

  export async function applyChargebackToReserve(
    userId: string,
    chargebackId: string,
    salesOrderId: string,
    chargebackAmountCents: number,
    chargebackType: 'VOLUNTARY_CANCELLATION' | 'NON_PAY_DISCONNECT',
    providerName: string,
    txDb?: TxDb
  ): Promise<{
    deductedFromReserveCents: number,
    remainingChargebackCents: number,
    newReserveBalance: number,
    reserveDeficit: boolean,
  }>
  
  Called when a chargeback is applied to an order.
  
  Steps:
  1. Load reserve for user
  2. Determine how much can come from reserve:
     const fromReserve = Math.min(
       chargebackAmountCents, 
       reserve.currentBalanceCents
     );
     const remaining = chargebackAmountCents - fromReserve;
  
  3. Create reserve_transaction:
       type: 'CHARGEBACK_DEDUCTION'
       amountCents: fromReserve
       isCredit: false
       balanceAfterCents: reserve.currentBalanceCents - fromReserve
       chargebackId
       salesOrderId
       description: `Chargeback deduction: ${providerName} 
                     ${chargebackType} — 
                     $${(chargebackAmountCents/100).toFixed(2)}`
  
  4. Update rolling_reserves:
       currentBalanceCents -= fromReserve
       totalChargebacksCents += chargebackAmountCents
       
       If newBalance < capCents AND status was 'AT_CAP':
         status = 'ACTIVE' (resume withholding)
         Create 'REPLENISHMENT' transaction:
         description: "Reserve dropped below cap due to 
                      chargeback. Withholding of 15% resumed."
       
       If remaining > 0:
         status = 'DEFICIT'
  
  5. If remaining > 0 (chargeback exceeded reserve):
     The remaining amount must come from future commissions.
     Create a negative adjustment on the next pay stub.
     Store in a new field on the chargeback record:
       excessAmountCents: remaining
     This will be deducted from the rep's next pay run 
     as a line item.
  
  6. Update salesOrders:
       carrierMaturityType = chargebackType
       maturityExpiresAt = calculateMaturityDate(
         order.dateSold, providerName, chargebackType
       )
  
  7. Log to audit_logs
  8. Return result

FUNCTION 6: handleSeparation

  export async function handleRepSeparation(
    userId: string,
    separationType: 'VOLUNTARY' | 'TERMINATED',
    appliedByUserId: string,
    txDb?: TxDb
  ): Promise<{
    reserveBalance: number,
    earliestReleaseDate: Date,
    ipadRecoveryApplied: boolean,
  }>
  
  Called when a rep is deactivated.
  
  Steps:
  1. Load reserve for user
  2. Set reserve.status = 'HELD'
  3. Set reserve.separatedAt = now()
  4. Set reserve.separationType
  
  5. Calculate earliestReleaseAt:
     -- Get all orders for this rep that are not yet mature
     -- Find the latest maturityExpiresAt date
     -- That date becomes the earliestReleaseAt
     -- If no immature orders: 
        earliestReleaseAt = now() + longest carrier maturity
     
     The policy states: reserve is held through full applicable
     maturity periods. Use the LONGEST maturity of any open order.
  
  6. If user.ipadIssued = true AND user.ipadReturnedAt IS NULL:
     Apply $500 equipment recovery automatically:
     await applyEquipmentRecovery(
       userId, 'IPAD', 50000, appliedByUserId,
       'iPad not returned at separation'
     )
     ipadRecoveryApplied = true
  
  7. Create reserve_transaction: type = 'CAP_REACHED' 
     with description:
     "Reserve held pending maturity period expiry. 
     Earliest release: ${earliestReleaseDate}. 
     No early release per policy."
  
  8. Notify ACCOUNTING and EXECUTIVE:
     "${repName} (${repId}) has separated (${separationType}). 
     Reserve of $${balance} held until ${earliestReleaseDate}. 
     No early release permitted per policy."
  
  9. Log to audit_logs: 'RESERVE_HELD_SEPARATION'
  
  10. Return result

FUNCTION 7: checkAndReleaseMaturedReserves

  export async function checkAndReleaseMaturedReserves(): Promise<{
    released: number,
    totalReleasedCents: number,
  }>
  
  Called by the daily scheduler.
  
  Logic:
  1. Find all reserves where:
     status = 'HELD'
     AND earliestReleaseAt <= now()
     AND separatedAt IS NOT NULL
  
  2. For each reserve:
     a. Check if any chargebacks are still pending 
        for this rep's orders (within maturity window)
     b. If no pending chargebacks and maturity has passed:
        Create reserve_transaction: type = 'RELEASE'
        amountCents: currentBalanceCents
        isCredit: false
        description: "Reserve released after maturity period. 
                      No outstanding chargebacks."
        Update reserve: status = 'RELEASED', 
                        currentBalanceCents = 0,
                        totalReleasedCents += amount
        Notify ACCOUNTING: "Reserve of $X released for 
                           ${repName} — maturity period complete"
  
  3. Return counts

  Title: IRON CREST SOLUTIONS LLC
  Subtitle: Chargebacks & Rolling Reserve Policy

  Section 1 — Rolling Reserve Requirement:
    Iron Crest Solutions LLC ("Iron Crest" or "Company") 
    maintains a rolling reserve account for each independent 
    sales representative ("Sales Rep") to offset potential 
    vendor chargebacks, cancellations, non-pay disconnects, 
    and commission reversals. Iron Crest will withhold fifteen 
    percent (15%) of all commissions otherwise payable to the 
    Sales Rep until the rolling reserve balance reaches two 
    thousand five hundred dollars ($2,500). Once the reserve 
    reaches $2,500, the 15% withholding will cease, subject 
    to this Policy. Sales Reps acknowledge the rolling reserve 
    is not wages and is a conditional holdback against future 
    liabilities.

  Section 2 — Chargebacks:
    Any vendor chargebacks, cancellations, non-pay disconnects, 
    clawbacks, or commission reversals attributable to a Sales 
    Rep's accounts shall be deducted directly from the rolling 
    reserve. Commissions are not fully earned or vested until 
    the applicable vendor chargeback maturity periods have expired.

  Section 3 — Replenishment of Rolling Reserve:
    If chargebacks reduce the rolling reserve below $2,500, 
    Iron Crest will resume withholding fifteen percent (15%) 
    of commissions until the reserve is restored. Iron Crest 
    has no obligation to advance commissions during a reserve 
    deficit.

  Section 4 — Adjustment of Rolling Reserve for Excessive 
  Chargebacks:
    If a Sales Rep generates chargebacks exceeding the standard 
    $2,500 reserve, Iron Crest may, in its sole and reasonable 
    discretion, increase the reserve cap, increase the 
    withholding percentage, or impose other risk controls. 
    Continued participation after notice constitutes acceptance.

  Section 5 — Vendor Chargeback Maturity Periods:
    Optimum: Voluntary Cancellation – 120 days; 
             Non-Pay Disconnect – 180 days.
    Astound: Voluntary Cancellation – 120 days; 
             Non-Pay Disconnect – 120 days.

  Section 6 — Company-Issued Equipment (iPad):
    If a Sales Rep is issued an iPad or other electronic device 
    for business use, such equipment remains the sole property 
    of Iron Crest Solutions LLC at all times. In the event the 
    Sales Rep voluntarily or involuntarily separates from the 
    Company, and the issued iPad is not returned or is returned 
    damaged beyond normal wear and tear, the full replacement 
    cost of the device ($500) shall be treated as a recoverable 
    charge. This amount will be deducted from the Sales Rep's 
    remaining rolling reserve balance in the same manner as 
    vendor chargebacks or other recoverable losses. If the 
    rolling reserve is insufficient, Iron Crest reserves the 
    right to set off the remaining balance against future 
    commissions or other amounts payable, where legally 
    permissible.

  Section 7 — Termination or Resignation:
    Upon resignation or termination for any reason, Iron Crest 
    will retain the rolling reserve through the full applicable 
    chargeback maturity period(s). If no chargebacks occur, 
    remaining reserve funds will be released after maturity. 
    No early release is required.

  Section 8 — Setoff Rights:
    Sales Reps authorize Iron Crest to set off any chargebacks, 
    refunds, reversals, penalties, or equipment recovery costs 
    against the rolling reserve, future commissions, or other 
    amounts payable.

  Section 9 — Governing Law and Venue:
    This Policy shall be governed by and construed in accordance 
    with the laws of the State of New Jersey, without regard to 
    conflict-of-law principles.

  Section 10 — Acknowledgment of Risk and Authority:
    Sales Reps acknowledge commission-based sales involve risk 
    and delayed reversals, and that acceptance of this Policy 
    is a condition of participation. Iron Crest retains final 
    authority regarding reserve administration and chargeback 
    determinations.

### TASK 10A — Update REP User Guide section on Rolling Reserve

Replace the existing rolling reserve description in the 
REP guide with this accurate version based on the policy:

ROLLING RESERVE — WHAT IT IS AND HOW IT WORKS

The rolling reserve is a financial protection account 
that Iron Crest holds on your behalf to cover potential 
chargebacks from carriers.

Here is how it works:

WITHHOLDING: 15% of your net commission on each order 
is held back and placed in your reserve account until 
the reserve reaches $2,500. After it reaches $2,500, 
no more is withheld unless a chargeback reduces it below 
that amount.

Example: You earn $200 on a 1 Gig sale. Your take-home 
pay is $170. The remaining $30 (15%) goes into your reserve.

CHARGEBACKS: If a customer cancels or does not pay their 
bill, the carrier takes the commission back from Iron Crest, 
and Iron Crest deducts it from your reserve. If the 
chargeback exceeds your reserve balance, the excess is 
deducted from your future commissions.

MATURITY PERIODS: Commissions are not fully yours until 
the carrier's maturity period passes without a chargeback:
  Optimum — Voluntary cancel: 120 days after the sale
  Optimum — Non-pay disconnect: 180 days after the sale
  Astound — Both types: 120 days after the sale

REPLENISHMENT: If chargebacks bring your reserve below 
$2,500, the 15% withholding automatically resumes until 
the reserve is back at $2,500.

WHEN YOU LEAVE: If you resign or are terminated, your 
reserve is held until all your open orders have passed 
their maturity periods. There is no early release.

EQUIPMENT: If you were issued an iPad and do not return 
it when you leave, $500 is automatically deducted from 
your reserve.

IMPORTANT: The rolling reserve is not your wages. It is 
a conditional holdback. You can view your reserve balance 
and full transaction history in the app under 
Earnings → Rolling Reserve.
