Using the existing payroll, AR, finance import, commission, and override 
infrastructure, implement a complete end-to-end automated payroll and 
accounting system for Iron Crest. The goal is a single coherent flow:

  Carrier pays Iron Crest (AR)
    → AR satisfies order
      → Order becomes payroll-ready
        → Pay run collects payroll-ready orders
          → Pay stubs generated per rep
            → Pay stubs finalized and locked
              → ACH/payment export generated
                → Accounting reconciliation confirmed

Do not rebuild existing infrastructure. Bridge the gaps between 
existing systems and automate the handoffs between each stage.

===================================================================
SECTION 1 — PAYROLL READINESS GATE
===================================================================

### CONCEPT
An order is "payroll-ready" when Iron Crest has confirmed it has 
received payment from the carrier for that order. This is determined 
by the AR system. Currently AR satisfaction updates order paymentStatus 
but does not formally signal payroll readiness.

### TASK 1A — Add payrollReadyAt field to salesOrders

Add to salesOrders table (Drizzle migration):
  - payrollReadyAt: timestamp nullable
  - payrollReadyTriggeredBy: varchar(20) nullable
    Values: 'AR_SATISFIED' | 'MANUAL_OVERRIDE' | 'AUTO_IMPORT'
  - payrollHoldReason: text nullable
    (populated when an order should be excluded from payroll 
    despite being approved — e.g. disputed, under review)
  - isPayrollHeld: boolean default false

### TASK 1B — Trigger payrollReadyAt automatically

In the following locations, when an order transitions to a state 
where Iron Crest has confirmed carrier payment, set payrollReadyAt:

LOCATION 1: AR payment recording (POST /api/finance/ar/:id/payments)
  When newStatus becomes 'SATISFIED':
    await storage.updateOrder(ar.orderId, {
      payrollReadyAt: new Date(),
      payrollReadyTriggeredBy: 'AR_SATISFIED',
    });

LOCATION 2: Finance import posting (POST /api/finance/imports/:id/post)
  When arStatus is set to 'SATISFIED' during the post operation:
    include payrollReadyAt: new Date() and 
    payrollReadyTriggeredBy: 'AUTO_IMPORT' in the order update

LOCATION 3: Manual override endpoint (new)
  POST /api/admin/orders/:id/payroll-ready
  Access: ADMIN and OPERATIONS only
  Body: { reason: string }
  Sets payrollReadyAt, payrollReadyTriggeredBy = 'MANUAL_OVERRIDE',
  creates audit log entry: 'ORDER_PAYROLL_READY_MANUAL'
  Use case: carrier confirmed payment outside the AR system

LOCATION 4: Payroll hold endpoint (new)
  POST /api/admin/orders/:id/payroll-hold
  Access: ADMIN and OPERATIONS only
  Body: { reason: string }
  Sets isPayrollHeld = true, payrollHoldReason = reason
  Creates audit log: 'ORDER_PAYROLL_HELD'

  POST /api/admin/orders/:id/payroll-release
  Access: ADMIN and OPERATIONS only
  Clears isPayrollHeld, clears payrollHoldReason
  Creates audit log: 'ORDER_PAYROLL_RELEASED'

### TASK 1C — Payroll readiness query

Add storage method:
  async getPayrollReadyOrders(filters?: {
    repId?: string,
    teamRepIds?: string[],
    serviceId?: string,
    providerId?: string,
    readyAfter?: string,
    readyBefore?: string,
    excludePayRunId?: string,
  }): Promise<SalesOrder[]>

  Returns orders where:
    payrollReadyAt IS NOT NULL
    AND isPayrollHeld = false
    AND approvalStatus = 'APPROVED'
    AND jobStatus = 'COMPLETED'
    AND paymentStatus != 'PAID'
    AND payRunId IS NULL (not already in a pay run)

This method becomes the SINGLE source of truth for what enters 
a pay run. Replace any existing pay run order collection logic 
with calls to this method.

SECTION 2 — AUTOMATED PAY RUN BUILDER
===================================================================

### TASK 2A — Pay run auto-builder endpoint

POST /api/admin/pay-runs/auto-build
Access: ADMIN and OPERATIONS only

Request body:
  {
    name: string,
    periodStart: string,        // YYYY-MM-DD
    periodEnd: string,          // YYYY-MM-DD
    repIds?: string[],          // optional — limit to specific reps
    includeOverrides: boolean,  // default true
    dryRun: boolean,            // if true, return preview without saving
  }

Actions:
  1. Call getPayrollReadyOrders with the provided filters
  2. Group orders by repId
  3. For each rep, calculate:
     a. All payroll-ready orders in the period
     b. All APPROVED override earnings with payRunId IS NULL
        for orders owned by this rep's upline recipients
        Wait — overrides go to RECIPIENTS not the rep who sold.
        Collect override earnings WHERE recipientUserId = upline user
        AND approvalStatus = 'APPROVED' AND payRunId IS NULL
     c. All active approved advances for the rep (for repayment)
     d. All active user deductions for the rep
     e. All approved bonuses/SPIFFs for the rep with payRunId IS NULL
  4. For each rep, compute pay statement totals:
     grossCommission: sum of baseCommissionEarned for their orders
     incentivesTotal: sum of incentiveEarned
     overrideEarningsTotal: sum of override earnings where 
                            recipientUserId = this rep's userId
     bonusesTotal: sum of approved bonuses
     chargebacksTotal: sum of active chargebacks
     deductionsTotal: sum of active user deductions
     advancesApplied: advance repayment amount due this period
     taxWithheld: calculated from userTaxProfile if exists
     netPay: grossCommission + incentivesTotal + overrideEarningsTotal 
             + bonusesTotal - chargebacksTotal - deductionsTotal 
             - advancesApplied - taxWithheld

  5. If dryRun = true: return preview without writing to DB
  6. If dryRun = false:
     a. Create a PayRun record
     b. Link all qualifying orders to the pay run
     c. Generate pay statements (see Section 3)
     d. Link override earnings to the pay run
     e. Return the created pay run with summary

### TASK 3B — Pay stub generator service

Create a new file: server/payStubGenerator.ts

Export a single function:
  async function generatePayStub(
    payRunId: string,
    userId: string,
    txDb?: TxDb
  ): Promise<PayStatement>

This function:

STEP 1 — Load all data for this rep in this pay run:
  const rep = await storage.getUserById(userId)
  const orders = await storage.getOrdersByPayRunId(payRunId) 
    filtered to orders where repId = rep.repId
  const overrideEarnings = await storage.getOverrideEarningsByPayRun(
    payRunId, userId)  // where recipientUserId = userId
  const chargebacks = await storage.getChargebacksByPayRun(payRunId)
    filtered to this rep
  const deductions = await storage.getActiveUserDeductions(userId)
  const advances = await storage.getActiveAdvancesForUser(userId)
  const taxProfile = await storage.getUserTaxProfile(userId)
  const bonuses = await storage.getPendingBonusesForPayRun(periodEnd)
    filtered to this rep
  const payRun = await storage.getPayRunById(payRunId)

STEP 2 — Generate stub number:
  const stubSeq = await storage.getNextStubSequence(payRunId)
  const stubNumber = `STUB-${periodYYYYMM}-${rep.repId}-${stubSeq.toString().padStart(3,'0')}`

STEP 3 — Calculate totals:
  Group orders by service type for the breakdown section.
  For each order, capture: invoiceNumber, customerName, dateSold,
  serviceDescription, providerName, installDate, baseCommissionEarned,
  incentiveEarned, overrideDeduction, repRoleAtSale

  grossCommission = sum of baseCommissionEarned across all rep's orders
  incentivesTotal = sum of incentiveEarned
  overrideEarningsTotal = sum of override earning amounts
  bonusesTotal = sum of bonus amounts
  chargebacksTotal = sum of chargeback amounts (stored as positive, 
                     subtracted in netPay calc)
  deductionsTotal = sum of deduction amounts
  
  advancesApplied = calculate repayment amount:
    For each active advance in REPAYING status:
      repaymentAmount = min(advance.remainingBalance, 
                           advance.repaymentAmountPerPeriod)
    advancesApplied = sum of all repaymentAmounts
  
  taxWithheld = 0 (default — calculate if taxProfile.withholding exists)
  
  netPay = grossCommission + incentivesTotal + overrideEarningsTotal 
           + bonusesTotal - chargebacksTotal - deductionsTotal 
           - advancesApplied - taxWithheld

STEP 4 — Create the PayStatement record:
  await storage.createPayStatement({
    payRunId,
    userId,
    repName: rep.name,
    repId: rep.repId,
    repRole: rep.role,
    repEmail: rep.email,
    stubNumber,
    periodStart: payRun.periodStart,
    periodEnd: payRun.periodEnd,
    grossCommission: grossCommission.toFixed(2),
    incentivesTotal: incentivesTotal.toFixed(2),
    overrideEarningsTotal: overrideEarningsTotal.toFixed(2),
    bonusesTotal: bonusesTotal.toFixed(2),
    chargebacksTotal: chargebacksTotal.toFixed(2),
    deductionsTotal: deductionsTotal.toFixed(2),
    advancesApplied: advancesApplied.toFixed(2),
    taxWithheld: taxWithheld.toFixed(2),
    netPay: netPay.toFixed(2),
    totalOrders: orders.length,
    totalConnects: orders.filter(o => o.jobStatus === 'COMPLETED').length,
    totalMobileLines: orders.reduce((s,o) => s + (o.mobileLinesQty||0), 0),
    status: 'DRAFT',
    isViewableByRep: false,
    companyName: 'Iron Crest',
  }, txDb)

STEP 5 — Create line items (one per order):
  For each order in rep's orders:
    await storage.createPayStatementLineItem({
      payStatementId: statement.id,
      category: 'Commission',
      description: `${serviceDescription} - ${customerName}`,
      sourceType: 'sales_order',
      sourceId: order.id,
      salesOrderId: order.id,
      invoiceNumber: order.invoiceNumber,
      customerName: order.customerName,
      dateSold: order.dateSold,
      serviceDescription: serviceName,
      providerName: providerName,
      installDate: order.installDate,
      repRoleAtSale: order.repRoleAtSale,
      quantity: 1,
      rate: order.baseCommissionEarned,
      amount: order.baseCommissionEarned,
    })

  For each override earning:
    await storage.createPayStatementLineItem({
      payStatementId: statement.id,
      category: 'Override',
      description: `${overrideEarning.overrideType} - ${orderInvoiceNumber}`,
      sourceType: 'override_earning',
      sourceId: overrideEarning.id,
      salesOrderId: overrideEarning.salesOrderId,
      invoiceNumber: linkedOrder.invoiceNumber,
      customerName: linkedOrder.customerName,
      dateSold: linkedOrder.dateSold,
      amount: overrideEarning.amount,
    })

  For each chargeback:
    Create line item with category: 'Chargeback', 
    amount: negative value

  For each deduction:
    Create line item with category: 'Deduction',
    amount: negative value

  For each bonus:
    Create line item with category: 'Bonus',
    link bonus to pay run via storage.linkBonusToPayRun()

  For each advance repayment:
    Create line item with category: 'Advance Repayment',
    amount: negative repaymentAmount
    Call storage.createAdvanceRepayment() to record it

STEP 6 — Return the complete statement

### TASK 3C — Batch pay stub generation endpoint

POST /api/admin/pay-runs/:id/generate-stubs
Access: ADMIN and OPERATIONS only

Actions:
  1. Verify pay run exists and status is 'DRAFT' or 'OPEN'
  2. Get all unique repIds from orders linked to this pay run
  3. Get all unique recipientUserIds from override earnings 
     linked to this pay run
  4. Combine into unique set of userIds
  5. For each userId, call generatePayStub() in a transaction
  6. Return:
     {
       generated: number,
       failed: number,
       failedUserIds: string[],
       totalNetPay: number,
       statements: Array<{ userId, repName, stubNumber, netPay }>
     }

### TASK 3D — Pay stub finalization

POST /api/admin/pay-runs/:id/finalize
Access: ADMIN only

Actions:
  1. Verify all pay statements for this run are in DRAFT status
  2. Verify no pay statements have netPay < 0 (flag but don't block)
  3. Update all pay statements: status = 'ISSUED', isViewableByRep = true
  4. Update pay run: status = 'FINALIZED', finalizedAt = now()
  5. Mark all linked orders: paymentStatus = 'PAID', paidDate = today
  6. Process all advance repayments recorded during stub generation
  7. Send notification to each rep:
     "Your pay stub for {periodStart} - {periodEnd} is ready. 
     Net pay: ${netPay}"
  8. Create audit log: 'PAY_RUN_FINALIZED'
  9. Return pay run summary
